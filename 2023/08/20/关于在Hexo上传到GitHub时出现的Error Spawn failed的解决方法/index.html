关于在Hexo上传到GitHub时出现的Error: Spawn failed的解决方法

1.问题描述

如下图，之前也运到过这个错误，于是就搜索各种资料并决解了这个问题，但是并没有记录下来。所以最近又碰到这个问题了不知道怎么解决。不过终于啊在我的百般搜索下和对我的各种折磨下将这个问题又解决了，为了防止以后又出现这个问题，我就把这个写成博文。

这张图片就是在Hexo d的时候报错了，不管我怎么clean、gender、server、deploy就是一直报这个错误，我就在百度上搜索，就找到了这网站，它这个文章详解了Error: Spawn failed这个报错的深层原因，我也发在了博客中，但是对我的报错没有用。于是我就又在CSDN上找答案，看了很多文章，报的错都是这个，但是解决方法却很多不同。我也尝试了里面的很多种方法，但是都没什么用。

看了这么多文章总结一下，这个报错大概就是说本地的服务没有连接上github，可能是网络的问题，可能是端口号的问题，也可能是一些配置连接信息错误的问题，还可能是.ssh密钥的问题。我就是通过重新创建了一个新的github的.ssh密钥重新连接了一下解决的。说一下创建.ssh密钥的步骤。

2.创建.ssh密钥的步骤

1、用git bash输入ssh-keygen -t rsa命令生成密钥

然后就敲回车确认创建文件位置，在敲回车确认创建空密码，最后敲回车确认密码，密钥就成功生成了。这里说明一下确认创建空密码的意思是未来怕你忘了这个密码，所以直接使用空密码。

2、找到电脑C盘>用户>当前登录的账户名>.ssh文件

这时候你的.ssh文件中只有下面两个文件，而这个config文件是后面要创建的。

3、打开id_rsa.pub文件，全选复制里面的内容，然后到github中的SSH and GPG keys 新建一个SSH key，再把复制的内容粘贴进去。这样密钥就成功创建了。

4、用git bash输入ssh -T git@github.com命令测试密钥是否正常，如果会出现

Hi XXX You’ve successfully authenticated, but github does not provide shell access.

就说明成功了，这时进行上传是没有问题的。

否则出现

ssh:connect to host github.com port 22: Connection timed out

说明连接还是有问题，这时就要创建一个config文件了，往下看。

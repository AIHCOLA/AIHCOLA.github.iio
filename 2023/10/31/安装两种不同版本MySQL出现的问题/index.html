安装两种不同版本MySQL出现的问题

一、第一次安装。

安装步骤

第一步-从官网下载MySQL

地址：https://dev.mysql.com/downloads/installer/

MySQL官网提供了三种安装方式，第一种是程序安装包联网在线安装（需联网），第二种是程序安装包本地安装（无需联网），第三种是压缩包本地安装。

这里讲解第三种安装方式——压缩包本地安装，第一次安装MySQL5版本。

第二步-解压指定的位置

下载好压缩包后，解压到你想要存放的位置，我的存放在了C盘的Program Files文件夹下的MySQL文件夹下。

第三步-配置系统环境变量

Win+Q搜索“查看高级系统设置”，点击进入“环境变量”，在系统变量中添加变量名为“MYSQL5_HOME”,变量值为“MySQL解压的路径”。注意这里的变量名中的5是为了区分安装其他不同版本，因为这里不能出现两个相同的变量。我现在安装的是5版本。

再双击进入到Path中配置MySQL的bin目录，点击新建，再点击浏览，找到MySQL的bin文件目录路径，复制粘贴过来就可以了，最后点击确定，MySQL5版本的系统环境变量就配置好了。

第四步-创建my.ini文件

环境配置完成后，打开MySQL文件夹，创建一个新的文件后缀名为 .ini 的 my.ini空白文件（如上图），将下面内容复制粘贴到my.ini文件中，更改MySQL的安装目录和数据库的数据存放目录为自己的MySQL的路径，注意\data不能丢。

第五步-初始化MySQL

Win+Q 搜索cmd，以管理员的身份运行cmd命令窗口，通过cmd命令进入到MySQL文件夹下的bin文件下，输入mysqld -initialize命令回车进行初始化。

然后回到MySQL文件夹下，会出现data文件夹，此文件不能手动创建，只能由系统生成。进入data文件夹，找到后缀为.err的文件，打开这个文件，找到root@localhost：后面就是初始密码。初始用户名默认是root。记住这个初始密码和初始用户名。

第六步-安装MySQL

在cmd命名中，输入mysqld --install 命令回车，当出现

Service Successfully installed

就安装成功了。

第七步-启动MySQL服务

开启MySQL服务，cmd命令窗口输入net start mysql,此时会出现

MySQL服务正在启动….

MySQL服务启动成功。

如果你需要修改密码，就要进行第八步，如下。

第八步-修改密码

要修改密码就要登入MySQL5，在cmd命令中，输入mysql -uroot -p命令回车，会出现

Enter password：

再输入初始密码，进入MySQL，输入alter user 'root'@'localhost' identified with mysql_native_password by '这里填写新密码';回车就修改密码成功了。

这是第一次安装，至此结束。

二、第二次安装

用压缩包本地安装，第二次安装MySQL8版本。

第一步，第二步同上

两个不同版本的MySQL最好放在两个文件夹下，只是我的MySQL8版本放在了E盘的MySQL8.0文件夹下。

第三步-配置MySQL8的环境变量

在配置环境变量之前，首先要到服务中查看MySQL5版本是否在运行，为了防止不必要麻烦，停止MySQL5服务，再为MySQL8添加环境变量。

再双击进入到Path中配置MySQL8的bin目录，点击新建，再点击浏览，找到MySQL8的bin文件目录路径，复制粘贴过来就可以了，这时为了防止冲突，可以先将MySQL5版本的环境变量路径删除，最后点击确定，MySQL8版本的系统环境变量就配置好了，等后面两个版本的MySQL都安装好了，再恢复MySQL5版本的环境变量。

此时如果没有删除MySQL5的环境变量，可能会导致MySQL8的注册表中的路径出现问题，最后导致开启服务报错。

发生系统错误 2

系统找不到指定文件。

第四步-创建MySQL8的my.ini文件

环境配置完成后，打开MySQL8文件夹，为MySQL8创建一个新的文件后缀名为 .ini 的 my.ini空白文件（如上图），将下面内容复制粘贴到my.ini文件中，更改MySQL8的安装目录和数据库的数据存放目录为自己的MySQL8的路径外，还要将端口号更改为3308。因为一个端口号只能被一个应用程序使用，如果另一个应用程序试图使用同一端口号，将会导致冲突并且其中一个应用程序将无法正常工作。因此，在配置网络应用程序时，需要确保每个应用程序使用不同的端口号，以避免冲突。而MySQL5已经占用了3306的端口号，所以MySQL8只能使用其他端口，我这里设置了3308，你们也可以设置其他没有使用的。

这时就有疑问了，那如果3308这个端口号也被其他应用程序使用了呢？我怎么知道哪个是没有被使用的？

端口号如果被其他占用再登录进入MySQL还会报

ERROR 1043 (08S01): Bad handshake 错误

那这时候那就要用到cmd命令netstat -ano，用来查看当前电脑中所有的端口号的使用情况，查找到占用端口的应用程序，将它关闭服务或者卸载就可以了。也可以用netstat -ano |findstr 3308 命令直接查看3308端口号的使用情况。

如果缺乏my.ini文件可能会报

MySQL 服务正在启动….

MySQL服务无法启动，

这种情况就要创建my.ini文件，卸载再重新安装后，启动服务。

第五步-初始化MySQL8同上

在初始化的过程中可能会出现使用了mysqld -initialize命令，但是却没有data文件的情况，这时可以使用mysqld --no-defaults --initialize命令回车重新生成data文件。

第六步-安装MySQL8

除了mysqld --install的安装命令，

还可以在命令行输入 mysql --install MySQL8命令(就是mysql --install 更改的服务名称)。这个命令可以用来更改MySQL服务的名称，用于区分两个不同的版本。而mysqld --install是默认的安装名为MySQL服务的命令。

还可以在命令行输入mysqld --install MySQL8 --defaults-file=‪E:\MySQL8.0\mysql-8.0.34-winx64\my.ini(名字命名为MySQL8,并且*指定默认的配置文件为刚才创建的my.ini文件*)。这个命令可以修正注册表中image Paths文件的路径问题。

所以在MySQL8的安装过程中为了区分两个版本的MySQL，也防止后续的冲突报错，在cmd命名中，输入mysql --install MySQL8命令回车，当出现

Service Successfully installed

就安装成功了。这个时候，PC中就有两个版本的MySQL了，一个名为MySQL5，另一个名为MySQL8。

第七步-启动并登录MySQL8服务

使用 net start mysql、net stop mysql 时报

服务正在启动或停止中，请稍候片刻后再试一次。

解决办法：打开任务管理器，找到 mysqld.exe，然后右击结束任务即可。

当安装了两个版本的MySQL之后，登录的命令就不再使用mysql -uroot -p，而用mysql -uroot -p -P3308(就是mysql -uroot -p -P端口号)回车就可以进入MySQL8数据库中。

第八步-修改密码同上

第九步-删除MySQL服务

cmd中输入sc delete 服务名就可以删除MySQL服务。

三、报错总结-解决方法

问题1：没有删除MySQL5的环境变量，导致MySQL8的注册表中的路径出现问题，最后导致开启服务报错。

问题2：端口号被其他应用程序占用导致报错。

问题3：3.1缺乏my.ini文件

​ 3.2缺乏data文件

​ 3.3端口号被占用

问题4：使用 net start mysql、net stop mysql 命令不起作用，报错。

问题5：使用了mysqld -initialize命令，但是却没有data文件的情况。
